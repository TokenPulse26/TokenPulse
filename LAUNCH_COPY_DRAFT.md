# TokenPulse Launch Copy Draft

## One-paragraph version

TokenPulse is a local-first proxy and dashboard that helps you see what your AI tools are actually doing across cloud and local models. Point a provider or tool at TokenPulse, send a request, and immediately see the provider, model, usage, cost, and request activity in one local dashboard you control.

---

## Short version

TokenPulse gives you one local dashboard for AI usage across cloud and local models.

---

## Slightly stronger version

If you are using a mix of Claude, OpenAI, OpenRouter, Ollama, or other AI tools, TokenPulse helps you see what is actually happening. It sits in front of your providers, tracks request activity locally, and gives you one dashboard for usage, cost, and visibility across your stack.

---

## Early-access truth version

TokenPulse is in early access. The strongest current path is a local technical setup where you point one route at TokenPulse, verify it in the dashboard, and expand from there. Ollama is currently the strongest verified local-model path. LM Studio route support exists, but it still carries lighter verification confidence today.

---

## What makes it interesting

- one local dashboard instead of scattered provider tabs
- visibility into what your AI stack is actually doing
- local-first request logging and usage tracking
- hybrid cloud + local model story in one place
- a believable proof-of-life flow instead of vague analytics claims

## Honest limitations block

Current early-access limitations:
- best suited for technical early adopters right now
- installer flow is still narrower than a polished mainstream release
- some provider paths are more strongly verified than others
- LM Studio support exists, but it is currently less verified than Ollama
