# TokenPulse Screenshot and Demo Checklist

## Goal

Create the smallest set of launch assets that make TokenPulse feel real quickly.

---

## Primary screenshot goal

The best screenshot should communicate all of this at a glance:
- real request activity
- clear provider/model visibility
- spend or usage signal
- enough polish to feel like a product, not an internal dev panel

### Best screenshot target
Capture a dashboard state that shows:
- summary cards with non-trivial values
- recent activity with recognizable provider/model rows
- provider/model breakdown visible in the same frame if possible
- a clean dark-theme product feel

Avoid screenshots that look:
- empty
- overly debuggy
- too sparse to feel real
- full of obviously test-only junk

---

## Recommended demo flow

### Demo story
1. Start TokenPulse
2. Open the dashboard
3. Send one real request through a known route
4. Refresh or show the dashboard updating
5. Point out the provider, model, and usage/cost visibility
6. If available, contrast with a second route or local-model example

### Best first route
- OpenAI-compatible or Anthropic for the cleanest “it works” moment

### Best optional second route
- Ollama, if you want to emphasize hybrid local + cloud visibility

Do not make LM Studio the hero route in the first public demo.

---

## Demo talking points

Use simple points:
- "This sits between your AI tools and providers."
- "Everything stays local-first."
- "You can actually see what requests are happening."
- "You get one dashboard across cloud and local usage."
- "The goal is visibility and trust, not another vague billing page."

---

## Pre-capture checklist

Before taking screenshots or recording:
- make sure the dashboard is populated with decent-looking data
- avoid obviously misleading or junk test rows if possible
- make sure the docs/launch wording nearby match current reality
- do not imply LM Studio is equally verified with Ollama
- keep the framing early-access if the polish is still uneven

---

## Minimum launch asset set

If time is tight, Monday only needs:
1. one strong screenshot
2. one short product paragraph
3. one short early-access disclaimer
4. one simple demo sequence

That is enough for a credible soft launch.
